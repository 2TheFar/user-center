# 前端开发报告：给用户中心补一个清爽的 Vue 前端

这次前端的目标很明确：不要把一个简单用户中心做成复杂后台，只把后端已经有的注册、登录、退出登录、个人资料维护、用户查询、用户删除和邀请注册能力接起来，页面干净一点，代码也别太绕。

2026-05-30 又做了一轮优化：参考 CodeFather Vue 前端教程的功能路线和页面组织思路，但没有照搬教程里的手写实现；具体代码按当前项目的 Vue 3、Vite、TypeScript、Ant Design Vue、Axios、Pinia 和 Vue Router 结构来落地。

## 技术选型

前端使用 Vue 3、Vite、TypeScript、Ant Design Vue、Axios、Pinia 和 Vue Router。工程层面补了 ESLint 和 Prettier，保证后面继续写页面时不容易把风格写散。

整个前端没有引入 Mock 数据，所有业务请求都走真实后端接口。开发环境里通过 Vite 代理把 `/api` 转发到 `http://localhost:8080`，避免浏览器直接从 `127.0.0.1:5173` 请求 `localhost:8080` 时出现跨域问题。

## 页面结构

目前页面分成四块：

- 登录页：账号、密码输入，成功后把当前用户保存到 Pinia；管理员进入用户管理页，普通用户进入我的资料页。
- 注册页：账号、密码、确认密码、邀请码输入，做必填、长度、账号字符、两次密码一致和邀请码格式校验。
- 我的资料页：普通用户的默认入口，可以修改昵称、头像、性别、手机号和邮箱。
- 用户管理页：管理员专属入口，包含用户统计、当前账号信息、查询区域、用户表格、删除确认弹窗，以及管理员邀请码工具区。

顶部导航后来单独抽成了 `AppHeader.vue`。最新版本改成了更接近正式应用的浅色粘性导航，当前用户区域展示头像、账号和角色。普通用户只看到“我的资料”入口，管理员同时看到“用户管理”和“我的资料”。登录和注册页也在 `AuthShell.vue` 里放了一个轻量顶栏，这样用户可以在登录、注册、资料页和管理页之间直接跳转，不用依赖浏览器返回。

## 接口接入

已接接口如下：

- `POST /user/register`
- `POST /user/login`
- `GET /user/current`
- `POST /user/profile/update`
- `POST /user/logout`
- `GET /user/search?username=xxx`
- `POST /user/delete`
- `POST /register-code/generate`
- `GET /register-code/check?code=xxx`

删除接口有一个容易写错的点：请求体是数字 id 本身，不是 `{ id: 1 }`。前端已经按后端实际 Controller 写法处理。

邀请注册这块目前分成两个入口：注册页只填写邀请码，不放单独校验按钮；管理员用户管理页提供“生成邀请码”和“校验邀请码”两个操作。虽然界面上叫“邀请码”，但为了匹配后端接口，请求字段仍然是 `registerCode`。注册接口返回 `-7` 时，前端统一提示“邀请码无效或已被使用”。

后端现在已经提供退出登录接口和当前用户接口。前端点击“退出登录”会先调用 `POST /user/logout`，后端返回成功后再清理 Pinia 并跳回登录页；页面刷新或应用重新启动时会调用 `GET /user/current`，根据 Session 恢复 Pinia 中的 `currentUser`。

我的资料页对接 `POST /user/profile/update`。前端只提交现有 `User` 字段中允许普通用户主动修改的资料项：`username`、`avatarUrl`、`phone`、`email`、`gender`。保存成功后后端返回最新脱敏用户，前端立即写回 Pinia，所以顶部昵称和头像会同步变化。

头像功能已经从“手动填写图片 URL”改成“本地图片上传”。前端先裁剪和预览，点击“保存资料”时再上传头像，并把后端返回的 `/user/avatar/xxx.png` 写入资料更新请求。

前端请求层现在已经统一解包 `BaseResponse<T>`：`code === 0` 时返回业务 `data`，业务页面不再重复判断统一响应结构；未登录、无权限和网络错误由请求层给出统一提示。

## 路由与权限

`/` 会根据登录态和角色分流：未登录进入 `/login`，普通用户进入 `/profile`，管理员进入 `/users`。`/profile` 要求登录；`/users` 要求登录且必须是管理员。普通用户直接访问 `/users` 时会被前端路由守卫带回 `/profile`，后端用户管理接口仍然继续做管理员校验。

## 设计取向

视觉上没有做花哨的营销页，也没有堆很多假模块。登录注册页偏轻量入口，资料页偏普通用户自助编辑，用户管理页偏普通后台系统：清晰头部、统计卡片、当前账号信息、查询区域、表格和操作按钮。

最新版本使用更克制的青色主色、浅色导航和白色内容面板，整体更像即将上线的正式应用服务，而不是测试页面。界面文案也已经改成普通用户能理解的表达，不再展示 `后端`、`Session`、`Spring Boot`、`前端状态`、`脱敏字段` 这类开发者视角词汇。

浏览器中也确认过 `/users`、`/login`、`/register` 都能正常渲染，`/api/user/search` 能通过 Vite 代理访问后端。2026-05-30 前端优化完成后，也再次通过 `npm run lint` 和 `npm run build` 做过类型、模板和构建检查。
