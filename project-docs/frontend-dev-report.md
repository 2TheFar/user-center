# 前端开发报告：给 Spring Boot 用户中心补一个清爽的 Vue 页面

这次前端的目标很明确：不要把一个简单用户中心做成复杂后台，只把后端已经有的注册、登录、退出登录、用户查询、用户删除和注册码接口接起来，页面干净一点，代码也别太绕。

## 技术选型

前端使用 Vue 3、Vite、TypeScript、Ant Design Vue、Axios、Pinia 和 Vue Router。工程层面补了 ESLint 和 Prettier，保证后面继续写页面时不容易把风格写散。

整个前端没有引入 Mock 数据，所有业务请求都走真实后端接口。开发环境里通过 Vite 代理把 `/api` 转发到 `http://localhost:8080`，避免浏览器直接从 `127.0.0.1:5173` 请求 `localhost:8080` 时出现跨域问题。

## 页面结构

目前页面分成三块：

- 登录页：账号、密码输入，成功后把当前用户保存到 Pinia，并跳转用户管理页。
- 注册页：账号、密码、确认密码、注册码输入，做最基本的必填、两次密码一致和注册码格式校验。
- 用户管理页：顶部搜索、用户表格、安全字段展示、删除确认弹窗，以及管理员注册码工具区。

顶部导航后来单独抽成了 `AppHeader.vue`，用户管理页使用蓝底白字的后台式导航。登录和注册页也在 `AuthShell.vue` 里放了一个轻量顶栏，这样用户可以在登录、注册、用户管理之间直接跳转，不用依赖浏览器返回。

## 接口接入

已接接口如下：

- `POST /user/register`
- `POST /user/login`
- `POST /user/logout`
- `GET /user/search?username=xxx`
- `POST /user/delete`
- `POST /register-code/generate`
- `GET /register-code/check?code=xxx`

删除接口有一个容易写错的点：请求体是数字 id 本身，不是 `{ id: 1 }`。前端已经按后端实际 Controller 写法处理。

注册码这块目前分成两个入口：注册页只填写注册码，不放单独校验按钮；管理员用户管理页提供“生成注册码”和“校验注册码”两个操作，方便手动测试后端接口。注册接口返回 `-7` 时，前端统一提示“注册码无效或已被使用”。

后端现在已经提供退出登录接口，前端点击“退出登录”会先调用 `POST /user/logout`，后端返回成功后再清理 Pinia 并跳回登录页。后端仍然没有当前用户接口，所以页面刷新后无法可靠恢复 Pinia 中的当前用户信息；页面文案会明确提示“仅以后端 Session 为准”。

## 设计取向

视觉上没有做花哨的营销页，也没有堆很多假模块。登录注册页偏轻量入口，用户管理页偏普通后台系统：清晰头部、内容区、搜索框、表格和操作按钮。

顶部导航改成了蓝底白字，主色和 Ant Design Vue 默认蓝保持一致。页面背景仍然是很浅的蓝灰色，内容区域使用白色面板，整体保持干净、好读。

## 验证结果

本次验证过：

```powershell
cd frontend
npm run lint
npm run build
```

两条命令都已经通过。构建时会提示主包体积偏大，这是 Ant Design Vue 整体引入后比较常见的体积提醒，不影响当前运行。后面如果要继续优化，可以做更细的组件按需加载和 chunk 拆分。

浏览器中也确认过 `/users`、`/login`、`/register` 都能正常渲染，`/api/user/search` 能通过 Vite 代理访问后端。注册码前端改动完成后，也通过 `npm run build` 做过类型和模板检查。

## 后续可以做但这次没有做的事

最值得补的是后端 `GET /user/current` 接口，用来恢复登录态。现在管理员页的注册码工具区依赖前端保存的当前用户状态，刷新页面后如果 Pinia 丢失，就需要重新登录管理员账号。

另一个可选方向是优化构建体积。目前页面功能很小，优先级不高；等页面变多以后再集中拆包更合适。
