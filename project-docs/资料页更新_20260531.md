# 资料页更新日志 2026-05-31

这次更新把普通用户的入口从“用户管理”调整为“我的资料”。用户管理继续保留给管理员使用，普通用户以后只维护自己的基础资料。

## 后端更新

- 新增 `POST /user/profile/update`，用于修改当前登录用户自己的资料。
- 新增 `UserUpdateProfileRequest`，请求体只接收 `username`、`avatarUrl`、`phone`、`email`、`gender`。
- 资料更新时从 Session 的 `userLoginState` 获取当前用户 ID，不使用前端传入的用户 ID。
- 保存成功后重新查询用户，复用 `getSafeUser` 返回脱敏用户，并把最新用户信息写回 Session。
- 字符串字段会先去掉前后空格，空字符串会转换为 `null`，方便用户清空可选资料。

## 前端更新

- 新增 `/profile` 页面，展示和编辑当前登录用户的基础资料。
- 登录成功后按角色分流：管理员进入 `/users`，普通用户进入 `/profile`。
- `/users` 增加前端路由守卫，普通用户直接访问时会回到 `/profile`。
- 顶部导航按角色展示入口：普通用户只显示“我的资料”，管理员显示“用户管理”和“我的资料”。
- 资料保存成功后，把后端返回的最新用户写回 Pinia，顶部昵称和头像会同步更新。

## 字段与校验

本次不新增数据库字段，只使用现有 `User` 属性中适合普通用户主动维护的字段：

- `username`：昵称，可为空，最多 256 个字符。
- `avatarUrl`：头像 URL，可为空，最多 1024 个字符，非空时必须以 `http://` 或 `https://` 开头。
- `phone`：手机号，可为空，非空时需符合 `^1[3-9]\d{9}$`。
- `email`：邮箱，可为空，非空时需符合基础邮箱格式。
- `gender`：性别，可为空，非空时只能是 `0` 或 `1`。

账号、密码、角色、状态、创建时间、更新时间和逻辑删除标记不在资料页修改范围内。
