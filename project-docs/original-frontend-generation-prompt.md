# 原始前端生成提示词

> 说明：这是用户曾经提供给 AI、用于生成当前用户中心前端的原始提示词。本文按用户本次消息原样整理保存，便于后续追溯前端实现目标、技术选型和约束边界。

```text
你是资深前端工程师，请为当前 Spring Boot 用户中心项目编写一个前端页面。
这个计划书并不一定完全贴合已有后端代码，如果遇到和后端代码不符的，一律灵活改动，一定不能动后端代码，除非后端代码出错了就停止并进行报告。


项目目标：
实现一个简洁、美观、大众化的用户中心前端，不做复杂业务设计，不做过度预留，只完成和当前后端接口对应的基础功能。

技术选型：
- Vue 3
- Vite，不使用 Vue CLI
- TypeScript
- Ant Design Vue
- Axios
- Pinia
- Vue Router
- ESLint + Prettier

整体风格要求：
1. 页面风格优美、简约、清爽，适合普通后台/用户中心系统。
2. 色彩均衡，不要太个性化，不要花哨，不要二次元风格。
3. 使用 Ant Design Vue 组件完成页面布局和表单。
4. 页面结构清晰，代码规范，组件命名清楚。
5. 不要做过度预留设计，不要添加后端没有的复杂功能。
6. 不要伪造太多业务页面，只围绕登录、注册、当前用户、用户查询、用户删除这些基础功能实现。

后端接口要求：
请优先检查项目中的 Controller 代码，以实际后端接口为准。
如果无法自动确认，则按以下接口约定实现：

1. 用户注册
POST /user/register
Content-Type: application/json

请求体：
{
  "userAccount": "",
  "userPassword": "",
  "confirmPassword": ""
}

2. 用户登录
POST /user/login
Content-Type: application/json

请求体：
{
  "userAccount": "",
  "userPassword": ""
}

登录成功后，后端使用 Session + Cookie 保存登录态，所以 Axios 必须开启 withCredentials。

3. 查询用户列表
GET /user/search?username=xxx

说明：
username 是可选查询参数。
如果 username 为空，则查询全部用户。
该接口需要管理员登录态。

4. 删除用户
POST /user/delete
Content-Type: application/json

请求体是一个数字 id，例如：
1

注意：这里不是 { "id": 1 }，而是直接传数字。
该接口需要管理员登录态。

前端页面要求：

1. 登录页
- 包含账号、密码输入框。
- 登录成功后保存当前用户信息到 Pinia。
- 登录成功后跳转到用户管理页或首页。
- 登录失败给出简洁提示。

2. 注册页
- 包含账号、密码、确认密码输入框。
- 注册成功后提示成功，并跳转登录页。
- 做基本前端校验，例如不能为空、两次密码一致。
- 不要写复杂校验规则，核心校验仍以后端为准。

3. 用户管理页
- 顶部提供用户名搜索框。
- 使用表格展示用户列表。
- 表格字段只展示安全字段，例如 id、账号、用户名、头像、邮箱、手机号、性别、角色、状态、创建时间等。
- 不展示密码、逻辑删除字段等敏感字段。
- 提供删除按钮，点击后调用删除接口。
- 删除前使用确认弹窗。
- 删除成功后刷新列表。

4. 路由与状态管理
- 使用 Vue Router 管理登录页、注册页、用户管理页。
- 使用 Pinia 保存当前登录用户信息。
- 页面刷新后，如果有 current user 接口则尝试恢复登录态；如果没有，就保持简单处理，不要强行设计复杂权限系统。
- 需要管理员权限的页面，如果后端返回无权限或失败，给出提示即可。

5. Axios 封装
- 创建统一 request 实例。
- baseURL 使用 http://localhost:8080。
- 开启 withCredentials: true，以支持 Session Cookie。
- 统一处理响应和错误提示。
- 不要过度封装，不要写复杂拦截器逻辑。

6. 工程要求
- 使用 Vue 3 Composition API。
- 使用 TypeScript 类型。
- 代码结构清晰，推荐目录：
  src/api
  src/router
  src/stores
  src/views
  src/components
  src/types
- 页面能直接运行。
- 不要引入无关依赖。
- 不要写 mock 数据，除非只是为了空状态展示。
- 所有请求都要真实调用后端接口。

请直接生成完整可运行的前端项目代码，并保证接口路径、请求方式、请求体格式和后端完全一致。
```
